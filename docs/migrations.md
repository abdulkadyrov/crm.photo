# Миграции

Текущая миграционная точка: `js/data/migrations.js`.

Правила:

- каждая миграция получает ID вида `migration-001-name`;
- миграция не удаляет пользовательские данные;
- повторный запуск должен быть безопасным;
- перед критическим изменением сохраняется мета-резервная копия;
- результат пишется в журнал миграций, когда store журнала будет добавлен отдельным повышением `DB_VERSION`.

На 12.07.2026 добавлена baseline-миграция без изменения текущей базы.
