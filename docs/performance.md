# Производительность

Целевые ограничения:

- не отрисовывать тысячи карточек одновременно;
- использовать thumbnails в списках;
- оригиналы загружать только при открытии, экспорте и печати;
- тяжёлые операции переносить в workers;
- ZIP не загружать целиком в память при следующих этапах переписывания архивации;
- использовать batch-транзакции через `js/data/db.js`.

Первый шаг выполнен:

- добавлены batch helpers `bulkPutRecords` и `bulkDeleteRecords`;
- подготовлены worker entrypoints;
- добавлен checksum worker scaffold;
- добавлены тесты для checksum, manifest и безопасных путей.
- добавлен deterministic fixture factory для малого, среднего и большого наборов без хранения бинарных гигабайтов в git.
- добавлен preflight-сервис для проверки manifest, путей и SHA-256 до подключения к боевому импорту.
- добавлен object URL registry для постепенного перевода предпросмотров на гарантированное освобождение `URL.revokeObjectURL()`.
- добавлен list performance helper для постраничной загрузки, виртуального диапазона и стабильной сортировки.
