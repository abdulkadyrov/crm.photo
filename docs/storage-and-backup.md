# Хранилище и резервные копии

Добавлен раздел `Настройки -> Хранилище и резервные копии`.

Экран показывает:

- используемый объём браузерного хранилища;
- доступную квоту;
- статус `navigator.storage.persisted()`;
- объём медиа, документов и готовых работ по текущим метаданным;
- выбранную папку для ПК, если браузер поддерживает File System Access API.
- предупреждения, если полный экспорт давно не выполнялся, добавлено много файлов, квота заполнена больше чем на 80%, есть изменения после последнего полного экспорта или ожидает обновление.

Кнопка выбора папки работает только после явного действия пользователя. После выбора выполняется тест `.vakha-write-test`: создать, записать, прочитать, удалить.

Если `showDirectoryPicker()` недоступен, приложение показывает ZIP-fallback и не ломает интерфейс.

После успешного полного ZIP-экспорта приложение записывает дату последней резервной копии и количество файлов на момент бэкапа. Локальные данные при этом не удаляются.
